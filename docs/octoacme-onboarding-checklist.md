# OctoAcme — Team Member Onboarding Checklist

## Purpose
Provide a structured onboarding experience for new team members to accelerate their productivity and integration into OctoAcme projects.

## Pre-Day One (Manager/PM)
- [ ] Provide team member with access to project repositories
- [ ] Add to relevant communication channels (Slack, Teams, etc.)
- [ ] Schedule onboarding meetings with key stakeholders
- [ ] Assign an onboarding buddy or mentor
- [ ] Share project documentation and process guides

## Week One: Foundations
### Administrative Setup
- [ ] Complete HR and IT onboarding requirements
- [ ] Set up development environment (tools, IDE, access credentials)
- [ ] Configure Git and repository access
- [ ] Join team calendar and recurring meetings
- [ ] Review security and compliance training

### Project Context
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles & Personas](octoacme-roles-and-personas.md) guide
- [ ] Understand current project phase and objectives
- [ ] Review project One-pager and success metrics
- [ ] Meet with Project Manager for project overview

### Team Introductions
- [ ] Meet 1:1 with direct manager
- [ ] Attend first team standup
- [ ] Schedule coffee chats with each team member
- [ ] Meet Product Manager to understand product vision
- [ ] Connect with onboarding buddy for questions

## Week Two: Engagement
### Process Familiarity
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md) guide
- [ ] Understand branching strategy and PR workflow
- [ ] Review Definition of Done and quality standards
- [ ] Explore project board and current sprint backlog
- [ ] Learn about team's testing and CI/CD practices

### Initial Contributions
- [ ] Pick up first "good first issue" or small task
- [ ] Create first pull request following team conventions
- [ ] Participate in code review (as reviewer or reviewee)
- [ ] Attend sprint planning or backlog refinement
- [ ] Ask questions and document answers for future reference

### Communication & Collaboration
- [ ] Understand escalation paths and support channels
- [ ] Review [Risk Management & Communication](octoacme-risks-and-communication.md) guide
- [ ] Learn team's communication preferences and norms
- [ ] Understand stakeholder landscape and reporting cadence

## Week Three-Four: Integration
### Deeper Involvement
- [ ] Take on more complex work items
- [ ] Participate actively in sprint ceremonies
- [ ] Contribute to team discussions and decision-making
- [ ] Help identify risks or dependencies in your work
- [ ] Shadow or pair with experienced team members

### Knowledge Building
- [ ] Review architecture and design documentation
- [ ] Understand key system components and integrations
- [ ] Learn about production environment and deployment process
- [ ] Review incident response procedures
- [ ] Explore monitoring and observability tools

### Feedback Loop
- [ ] Schedule 30-day check-in with manager
- [ ] Provide feedback on onboarding experience
- [ ] Identify any documentation gaps or improvement opportunities
- [ ] Clarify role expectations and success criteria
- [ ] Set goals for first 90 days

## Role-Specific Additions

### For Developers
- [ ] Review codebase structure and coding standards
- [ ] Set up local development and test environment
- [ ] Complete first deployment to staging environment
- [ ] Review security scanning and code quality tools

### For Product/Project Managers
- [ ] Shadow current PM in stakeholder meetings
- [ ] Review historical project artifacts and retrospectives
- [ ] Understand budget and resource constraints
- [ ] Learn project reporting tools and templates

### For UX Designers
- [ ] Review design system and style guides
- [ ] Set up design tools (Figma, Sketch, etc.)
- [ ] Review previous user research and usability studies
- [ ] Understand accessibility requirements and standards

### For Scrum Masters
- [ ] Shadow current Scrum Master in ceremonies
- [ ] Review team velocity and historical sprint data
- [ ] Understand team's current impediments and challenges
- [ ] Learn organization's agile coaching resources

### For Business Analysts
- [ ] Review existing requirements documentation
- [ ] Understand business stakeholder landscape
- [ ] Learn requirements management tools and processes
- [ ] Shadow in requirements gathering session

### For Support Engineers
- [ ] Set up support ticketing system access
- [ ] Review knowledge base and common issues
- [ ] Shadow experienced support engineer
- [ ] Learn escalation procedures and SLAs
- [ ] Understand product features from support perspective

## 90-Day Milestone
- [ ] Fully contributing to team deliverables
- [ ] Comfortable with team processes and tools
- [ ] Building relationships across the organization
- [ ] Identifying and executing on improvement opportunities
- [ ] Ready to mentor future new team members

## Tips for Success
- **Ask questions early and often** — No question is too small
- **Document as you learn** — Help improve docs for future team members
- **Build relationships** — Schedule 1:1s with cross-functional partners
- **Be patient with yourself** — It takes time to ramp up fully
- **Seek feedback regularly** — Don't wait for formal reviews
